# Product Naming

Generate strong product names and validate them before you fall in love. This skill combines creative generation with rigorous real-world validation so you don't build a brand on a name you can't own.

## The Hard Truth About Naming in 2026

**Every real English word is taken.** The internet is 30+ years old. Every noun, verb, adjective, and metaphor has been registered as a domain, launched as a product, or squatted on by a broker. If you can find it in a dictionary, someone owns it.

The most successful modern brands understood this: **Spotify, Figma, Vercel, Twilio, Klarna, Asana, Trello, Airtable, Zapier, Supabase.** None are real words. All are memorable, spellable, and ownable.

Your job is to generate names that DON'T exist yet. Not find real words that happen to be available. They won't be.

## When to Use This Skill

- Naming a new product, app, or SaaS from scratch
- Renaming to avoid a competitor conflict
- Evaluating a shortlist of name candidates
- Checking if a name is truly clear before committing

## The Process

### Phase 1: Creative Brief (2 minutes)

Before generating names, extract these from the user or the context:

1. **What it does** -- one sentence, plain language
2. **Who it's for** -- target user in specific terms
3. **Core emotion** -- what should the name make someone feel? (trust, speed, warmth, power, simplicity)
4. **Competitive landscape** -- who are the main competitors and what do their names sound like?
5. **Constraints** -- any must-haves (e.g., ".com required", "must work in Spanish", "no more than 8 chars")
6. **Name style preference** -- which of the strategies below appeals?

### Phase 2: Generation

**CRITICAL: At least 70% of generated candidates MUST be invented/non-dictionary words.** Real words will almost always fail validation. Prioritize strategies 1-4 below. Use strategies 5-7 only as supplements.

Generate 5-8 candidates per strategy. Pick the 3-4 most relevant strategies based on the brief.

---

**Strategy 1: Invented/Coined (PRIMARY)** -- Made-up words with phonetic intent.

The gold standard for new brands. These are maximally ownable and trademarkable.

Examples and how they were built:
- **Spotify** = spot + ify (finding + action suffix)
- **Figma** = fig (figurative) + ma (making) -- punchy, 2 syllables
- **Vercel** = ver (version/velocity) + cel (accelerate) 
- **Twilio** = twi (twilight/twist) + lio (melodic ending)
- **Klarna** = klar (Swedish for "clear") + na (soft ending)
- **Trello** = trellis (support structure) shortened + o ending
- **Zapier** = zap + ier (like "happier" -- makes things zappier)

**Method -- The Phonetic Construction Kit:**

1. **Start with root sounds** from words related to your product. Extract 2-3 letter fragments:
   - "proof" -> pro, pru, pruf
   - "trust" -> tru, trus, trs
   - "praise" -> pra, prai, prz
   - "voice" -> vo, voi, vox
   - "endorse" -> en, dor, ors

2. **Combine fragments** with pleasing endings:
   - Soft/friendly endings: -a, -o, -ly, -io, -ia, -ey
   - Tech/sharp endings: -ik, -ix, -el, -al, -er
   - Action endings: -fy, -ze, -go

3. **Phonetic rules for feel:**
   - Hard consonants (K, T, P, B) = power, precision, tech
   - Soft consonants (L, M, N, S) = warmth, trust, ease
   - Vowel-heavy = approachable, friendly
   - Consonant clusters = technical, serious

4. **Test each candidate:** Say it 5 times fast. If it flows, keep it. If you stumble, kill it.

---

**Strategy 2: Smart Respelling** -- Take a relevant word and mutate it until it's unique.

NOT just dropping a vowel (that's tired). Real mutations:

Examples:
- **Lyft** = lift with y swap (energy, movement)
- **Fiverr** = fiver with doubled r (emphasis)
- **Dribbble** = dribble with tripled b (playful)
- **Tumblr** = tumbler minus e (truncation)

**Method -- The Mutation Toolkit:**
- **Vowel swap:** a->e, e->i, i->y, o->u (e.g., "proven" -> "pruven")
- **Consonant double:** emphasize a sound (e.g., "proof" -> "prooff")
- **Letter swap:** swap similar sounds k/c, f/ph, s/z (e.g., "praise" -> "praize")
- **Truncation:** chop the last 1-2 letters (e.g., "endorse" -> "endors")
- **Suffix graft:** chop ending, add new one (e.g., "vouch" -> "voucht", "vouchly")
- **Phonetic respelling:** spell it how it sounds in another accent (e.g., "proof" -> "pruf")

**Rules:** The original word must still be recognizable within 2 seconds. If people can't reverse-engineer the root, you've gone too far.

---

**Strategy 3: Portmanteau** -- Blend fragments of two words into one new word.

Examples:
- **Instagram** = instant + telegram
- **Groupon** = group + coupon
- **Pinterest** = pin + interest
- **Shopify** = shop + -ify
- **Airtable** = air + table

**Method:**
1. List 10 words related to the product's FUNCTION (collect, display, show, prove, wall, board)
2. List 10 words related to the product's BENEFIT (trust, growth, sales, convert, shine, glow)
3. Find splice points where end-sounds of word A flow into start-sounds of word B
4. The blend must create a NEW word, not two words jammed together
5. Good splice: sounds overlap (Insta-gram). Bad splice: sounds collide (Trust-Board = just two words)

---

**Strategy 4: Root + Invented Suffix** -- Meaningful root with a made-up or unusual ending.

Examples:
- **Shopify** = shop + ify
- **Sendbird** = send + bird
- **Supabase** = supa + base
- **Datadog** = data + dog
- **Postmark** = post + mark

**Method:**
1. Pick a strong 3-5 letter root from the product domain
2. Add an unexpected suffix that creates a new word: -ify, -bird, -mark, -base, -dot, -fox, -bit, -hub, -pad, -spark, -wave, -mint
3. The combination must not already exist as a product. Check before falling in love.

---

**Strategy 5: Compound (SUPPLEMENT ONLY)** -- Two real words fused.

WARNING: These almost always fail validation. Every good compound is taken. Only use if the combination is genuinely unusual.

Examples that worked (years ago): Dropbox, Snapchat, Mailchimp.

---

**Strategy 6: Metaphor (SUPPLEMENT ONLY)** -- Real word borrowed from another domain.

WARNING: Dictionary words are taken. Only use if the word is obscure enough to have slipped through 30 years of domain registration. Think nautical terms, musical terms, architectural terms, botanical terms -- not common nouns.

---

**Strategy 7: Action Verb (SUPPLEMENT ONLY)** -- The brand is a verb.

WARNING: All good verbs are taken. Only suggest if the verb is obscure or you're mutating it (Strategy 2).

---

### Phase 3: First Cut

From all generated candidates, shortlist 8-10 that pass the **gut check**:

- Can you say it in a noisy room and be understood?
- Can someone spell it after hearing it once? (For invented words: is the spelling intuitive from the sound?)
- Is it under 10 characters?
- Does it NOT sound like an existing major brand? (No "Spottify", "Figmo", "Vercal")
- Would you feel good saying "we use [name]" to a colleague?
- Does it look good written down? (No awkward letter combos like "xq" or "uu")

### Phase 4: Validation (Critical -- Do Not Skip)

For each shortlisted name, run ALL of these checks. Use WebSearch and WebFetch. Do not present unvalidated names to the user.

#### Check 1: Web Search
```
WebSearch: "[name]" (exact match)
WebSearch: "[name] app" OR "[name] software"
```
Look for: existing products with the same name. For invented words, an exact match is a dealbreaker. For near-matches (off by one letter), assess whether confusion is likely.

#### Check 2: Domain Probe
```
WebFetch: https://[name].com
WebFetch: https://[name].io
```
- **Connection refused / timeout / DNS error** = likely unregistered (good)
- **Parked page / "domain for sale"** = available but may cost money (acceptable if price is reasonable)
- **Live product** = taken (eliminate unless you can use a modifier like "get[name].com" or "use[name].com")

If .com and .io both fail, also try:
```
WebFetch: https://get[name].com
WebFetch: https://use[name].com
WebFetch: https://[name]app.com
```

#### Check 3: Product Hunt Search
```
WebSearch: "[name] site:producthunt.com"
```
Even dormant products on PH create confusion.

#### Check 4: Social Handle Scan
```
WebSearch: "@[name] twitter" OR "@[name] x.com"
```
Social handles don't need to be available (you can use "get[name]" or "[name]hq"), but check if a competitor holds them.

#### Check 5: Negative Connotation Screen
```
WebSearch: "[name] slang meaning" OR "[name] urban dictionary"
```
Especially important for invented words -- they might accidentally sound like something rude in another language or subculture.

### Phase 5: Scoring

Score each validated survivor on 8 dimensions (1-5 scale):

| Dimension | What It Measures | 5 = Best |
|-----------|-----------------|----------|
| **Memorability** | Recall after one hearing | Sticks instantly, 1-2 syllables |
| **Pronounceability** | Say it without hesitation | Obvious pronunciation, no ambiguity |
| **Spellability** | Type it from hearing it | One obvious spelling, even for invented words |
| **Distinctiveness** | Stand apart from competitors | Nothing sounds like it in the space |
| **Meaningfulness** | Evokes the product's benefit | Root words hint at purpose |
| **Emotional Resonance** | Triggers the right feeling | Matches the core emotion from brief |
| **Visual Appeal** | Looks good in logo/URL/icon | Good letter shapes, balanced length |
| **Ownability** | Can you actually own this? | Domain clear, no trademark conflicts, no similar products |

**Total: /40. Above 30 = strong. Above 35 = exceptional.**

### Phase 6: Recommendation

Present the top 3 candidates in a table:

```
| Rank | Name | Score | Domain | Strategy | Why It Works |
|------|------|-------|--------|----------|--------------|
| 1    | ...  | /40   | .com?  | Type     | ...          |
| 2    | ...  | /40   | .com?  | Type     | ...          |
| 3    | ...  | /40   | .com?  | Type     | ...          |
```

Include a clear recommendation with reasoning. If no candidate scores above 30, go back to Phase 2 and generate more.

## Anti-Patterns

- **Never suggest real dictionary words as primary candidates.** They are taken. All of them. Use them only as roots for mutation.
- **Never present unvalidated names.** Every name shown to the user must have passed all 5 validation checks.
- **Never suggest names with >3 syllables** unless the brand demands gravitas.
- **Never suggest acronyms** for new/unknown products. You haven't earned the abbreviation yet.
- **Never clone a famous brand's pattern.** No "[Word]ify" if Shopify exists. No "[Word]ly" if Grammarly exists. No "[Word]base" if Supabase exists. Find your own suffix.
- **Don't fall in love before checking.** The graveyard of startups is full of beautiful names that belonged to someone else.

## Speed Run (When User Already Has Candidates)

If the user has names they want checked, skip straight to Phase 4 (Validation) and Phase 5 (Scoring). Don't generate new names unless asked.
